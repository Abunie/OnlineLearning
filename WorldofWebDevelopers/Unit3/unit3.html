XML

XML stands for eXtensible Markup Language. XML is a portable, widely supported open technology for data storage and exchange. It permits document authors to create markup fot virtually any type of information.It described daata in a way that human beings can understand and computers can process. An xml parser is responsible for identifying components in data structure for manipulation. Unlike html, xml does not have predefined tags so the author most define both the tags and the structure

Tree Structure

XML documents are formed as element trees. The tree starts from the root element and branches to the children. The terms used to discribe this relationship are parent, child , siblings. Below I have shown an example of an xml structure.

Some Important XML Rules
  • XML documents must contain one general root element that is the parent of all other elements
  • The XML prolog: <?xml version="1.0" encoding="UTF-8"?> is optional but if it must appear then it must be on the first line of the document
  • All XML elements must have a starting and closing tag
  • XML tags are case-sensitive.
  • XML elements must be properly nested
  • XML attribute values must always be quoted
  • Two dashes in the middle of a comment are not allowed e.g < !-- This is an invalid -- comment --</subchild>

AJAX

AJAX stands for Asynchronous JavaScript and XML. AJAX applications seperate client-side user interaction and server communication, and run them in parrallel, making the server communication and the run them in parrallel, making delays of server side processing more transparent to the user. A classic HTML registration form sends all of the data to be validated to the server when the user clicks the register button but Ajax-enabled forms are more interactive entries are validated individually, dynamically as the user enters data into the fields.
